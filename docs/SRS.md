# PsyForms

# Software Requirements Specification (SRS)

Version: 3.0 Final

Status: Approved

---

# 1. معرفی

PsyForms یک نرم‌افزار اندرویدی برای ایجاد، مدیریت، اجرا و ذخیره‌سازی پرسشنامه‌های روان‌شناختی است.

هدف پروژه، فراهم کردن بستری انعطاف‌پذیر برای توسعه‌دهندگان است تا بدون نیاز به برنامه‌نویسی بتوانند پرسشنامه‌های جدید را ایجاد و منتشر کنند.

---

# 2. نسخه‌های برنامه

برنامه دارای دو نسخه است.

## Developer

امکانات:

- ورود با نام کاربری و رمز عبور
- افزودن حیطه
- ویرایش حیطه
- حذف حیطه
- افزودن پرسشنامه
- ویرایش پرسشنامه
- حذف پرسشنامه
- افزودن خرده‌مقیاس
- ویرایش خرده‌مقیاس
- حذف خرده‌مقیاس
- افزودن سؤال
- ویرایش سؤال
- حذف سؤال
- تعریف گزینه‌های پاسخ
- تعریف نمره معکوس
- انتشار Repository
- پاسخ‌دهی به پرسشنامه‌ها
- خروجی Excel

---

## User

امکانات:

- مشاهده حیطه‌ها
- مشاهده پرسشنامه‌ها
- پاسخ‌دهی
- ذخیره جلسات
- خروجی Excel

در نسخه User هیچ امکانی برای ویرایش ساختار پرسشنامه وجود ندارد.

---

# 3. ساختار کلی

```
حیطه

    ↓

پرسشنامه

    ↓

خرده‌مقیاس

    ↓

سؤال
```

---

# 4. حیطه (Domain)

هر حیطه دارای:

- نام
- ترتیب نمایش

است.

نمونه:

- آسیب‌شناسی روانی کودک
- رفتاردرمانی
- ذهنی‌سازی

---

# 5. پرسشنامه

هر پرسشنامه دارای:

- نام
- توضیح پرسشنامه
- ترتیب نمایش
- وضعیت فعال/غیرفعال
- داشتن یا نداشتن نمره کل

است.

---

# 6. توضیح پرسشنامه

هر پرسشنامه دارای یک متن راهنما است.

این متن قبل از شروع پاسخ‌دهی نمایش داده می‌شود.

هدف آن:

- معرفی پرسشنامه
- دستورالعمل تکمیل
- توضیح شیوه پاسخ‌دهی

است.

---

# 7. خرده‌مقیاس

هر پرسشنامه شامل یک یا چند خرده‌مقیاس است.

هر خرده‌مقیاس دارای:

- نام
- توضیح کوتاه
- ترتیب

است.

---

# 8. توضیح خرده‌مقیاس

در ابتدای ورود به هر خرده‌مقیاس، صفحه‌ای نمایش داده می‌شود که شامل:

- عنوان خرده‌مقیاس
- توضیح خرده‌مقیاس

است.

کاربر سپس یکی از دو گزینه را انتخاب می‌کند:

- پاسخ می‌دهم
- برو به خرده‌مقیاس بعدی

---

# 9. رد کردن خرده‌مقیاس

اگر کاربر گزینه:

«برو به خرده‌مقیاس بعدی»

را انتخاب کند:

- تمام سؤال‌های آن خرده‌مقیاس رد می‌شوند.
- ثبت نهایی مجاز است.
- آن خرده‌مقیاس در نمره‌گذاری لحاظ نمی‌شود.

---

# 10. سؤال

هر سؤال یکی از انواع زیر است:

- پاسخ متنی
- پاسخ عددی
- چندگزینه‌ای

---

# 11. سؤال چندگزینه‌ای

در هنگام ایجاد سؤال چندگزینه‌ای، توسعه‌دهنده مشخص می‌کند:

- تعداد گزینه‌ها
- متن گزینه‌ها
- امتیاز هر گزینه

در صورت تکرار همان ساختار، می‌توان از قالب‌های قبلی استفاده کرد.

نامی برای قالب ذخیره نمی‌شود.

---

# 12. نمره‌گذاری معکوس

پس از پایان ورود سؤال‌های هر خرده‌مقیاس، توسعه‌دهنده یکی از دو گزینه را انتخاب می‌کند:

- این خرده‌مقیاس سؤال معکوس ندارد.
- وارد کردن شماره سؤال‌های معکوس

در صورت انتخاب گزینه دوم، شماره سؤال‌ها ثبت می‌شود.

---

# 13. نمره کل

پس از پایان تنظیم تمام خرده‌مقیاس‌ها، توسعه‌دهنده انتخاب می‌کند:

- این پرسشنامه نمره کل دارد.
- این پرسشنامه نمره کل ندارد.

---

# 14. جلسه (Session)

هر بار که آزمودنی پاسخ‌دهی را آغاز می‌کند، یک Session تشکیل می‌شود.

اگر چند پرسشنامه پشت سر هم تکمیل شوند، همگی متعلق به همان Session خواهند بود.

---

# 15. اطلاعات جمعیت‌شناختی

در ابتدای اولین پرسشنامه هر Session اطلاعات زیر دریافت می‌شود:

- جنسیت
- سن
- مقطع تحصیلی
- تحصیلات والدین

در همان Session دیگر تکرار نمی‌شود.

---

# 16. اعتبارسنجی

اگر کاربر وارد یک خرده‌مقیاس شده باشد:

تمام سؤال‌های همان خرده‌مقیاس باید پاسخ داده شوند.

در غیر این صورت ثبت نهایی مجاز نیست.

---

# 17. ناوبری سؤال‌ها

در نوار ناوبری:

- سؤال‌های تکمیل‌شده سبز
- سؤال جاری آبی
- سؤال‌های ناقص قرمز

نمایش داده می‌شوند.

اگر خرده‌مقیاسی ناقص باشد، عنوان آن نیز قرمز نمایش داده می‌شود.

---

# 18. ذخیره پاسخ‌ها

پس از ثبت موفق:

موارد زیر ذخیره می‌شوند:

- تاریخ
- ساعت
- اطلاعات جمعیت‌شناختی
- پاسخ سؤال‌ها
- نمره خرده‌مقیاس‌ها
- نمره کل (در صورت وجود)

---

# 19. تکرار پاسخ‌دهی

کاربر می‌تواند یک پرسشنامه را هر تعداد بار که بخواهد تکمیل کند.

---

# 20. خروجی Excel

دو نوع خروجی وجود دارد:

- خروجی جلسه کنونی
- خروجی همه جلسات

---

# 21. محل ذخیره فایل Excel

هنگام تولید فایل Excel، برنامه از کاربر می‌خواهد محل ذخیره فایل را انتخاب کند.

---

# 22. نام فایل Excel

نام فایل فقط شامل تاریخ و ساعت تولید خروجی است.

---

# 23. ساختار فایل Excel

فایل دارای دو Sheet است.

## Sheet اول

شامل:

- اطلاعات جمعیت‌شناختی
- نمره خرده‌مقیاس‌ها
- نمره کل پرسشنامه‌ها

هر ردیف مربوط به یک Session است.

اگر Session فقط شامل یک حیطه باشد، فقط همان حیطه در ردیف ثبت می‌شود.

اگر Session شامل چند حیطه باشد، داده‌ها به ترتیب حیطه‌ها در همان ردیف قرار می‌گیرند.

ستون‌های مربوط به حیطه‌هایی که در آن Session پاسخ داده نشده‌اند ایجاد نمی‌شوند.

ساختار ستون‌ها براساس Repository منتشرشده ثابت است.

---

## Sheet دوم

شامل:

- نمره تک‌تک سؤال‌ها

هر ردیف مربوط به یک Session است.

---

# 24. انتشار Repository

در نسخه Developer گزینه‌ای با عنوان:

«انتشار پرسشنامه‌ها»

وجود دارد.

پس از انتخاب آن:

برنامه فایل

psyforms_repository.json

را تولید می‌کند.

این فایل فقط شامل ساختار پرسشنامه‌ها است و شامل پاسخ کاربران نیست.

---

# 25. به‌روزرسانی APK

توسعه‌دهنده فایل

psyforms_repository.json

را در Repository گیت‌هاب جایگزین می‌کند.

GitHub Actions پروژه را Build می‌کند.

نسخه‌های جدید Developer و User تمام پرسشنامه‌های منتشرشده را در خود خواهند داشت.

---

# 26. امنیت

نسخه Developer با نام کاربری و رمز عبور محافظت می‌شود.

نسخه User فاقد بخش ورود است.

---

# 27. محدودیت‌ها

- حذف یا تغییر ساختار پرسشنامه فقط در نسخه Developer امکان‌پذیر است.
- پاسخ‌های کاربران هرگز در فایل Repository منتشر نمی‌شوند.
- فایل JSON فقط شامل ساختار پرسشنامه‌ها است.

---

# 28. اهداف طراحی

- سادگی استفاده
- توسعه بدون برنامه‌نویسی
- انعطاف‌پذیری
- نگهداری آسان
- پایداری داده‌ها
- قابلیت توسعه در آینده
